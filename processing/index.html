What is Generative Art?

According to Wikipedia...

Generative art refers to art that in whole or in part has been created with the use of an autonomous system.
Autonomous system: basically an algorithm, but could be any external source outside the direct control of the artist. A visualization of traffic flow patterns, for example. "Generative Art" is often used to refer to computer generated artwork that is algorithmically determined.

Applications: These tools provide a graphical interface for building and experimenting with visualizations.
vvvv (vier V; German)

Programming Libraries: There are a number of libraries and languages that help facilitate the creation of creative works. Cinder, and OpenFrameworks are both C++ libraries that provide a myriad of tools for building audio/visual applications. Processing, which we'll be looking at today, provides its own platform (on top of Java) and attempts to lower the barrier of entry for creating visual works.

What is Processing?

Processing is an open source programming language and environment for people who want to create images, animations, and interactions.

Processing is an open source programming language and environment for people who want to create images, animations, and interactions.

Initially, Processing was built as a sort of software "sketchbook", for teaching the fundamentals of programming.

Today, Processing is in use by thousands of professional for prototyping and creating production projects.
